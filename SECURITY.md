# Security Policy

Thanks for helping keep **Punds** safe. This is a small personal landing page —
a single deployed static site at [punds.ch](https://punds.ch/) with no backend,
no database, and no user accounts or user data. That keeps the attack surface
small, but reports are still genuinely appreciated.

## Supported Versions

Only the latest `main` is supported. The site is a single static deployment, so
whatever is currently live (built from `main`) is the only version that receives
fixes. There are no older releases to patch.

| Version       | Supported |
| ------------- | --------- |
| latest `main` | ✅        |
| anything else | ❌        |

## Reporting a Vulnerability

Please report security issues **privately** — do not open a public issue for
something exploitable.

You can report in either of these ways:

- **GitHub private security advisories** — go to the
  [Security tab](https://github.com/dominikkoenitzer/Punds/security/advisories)
  of the repository and click *Report a vulnerability*.
- **Email** — [dominik.koenitzer@gmail.com](mailto:dominik.koenitzer@gmail.com).

### What to include

To help me understand and reproduce the issue, please include where you can:

- A description of the vulnerability and its potential impact.
- Steps to reproduce, or a proof of concept.
- The affected page, URL, or file.
- Any relevant browser, version, or environment details.

## Response Expectations

This is a personal side project maintained in spare time, so please be patient:

- I aim to acknowledge a report within about **7 days**.
- If the issue is valid, I'll work on a fix and keep you posted on progress.
- Once it's resolved, I'm happy to credit you if you'd like.

Please give me a reasonable chance to address the issue before disclosing it
publicly. Thanks for reporting responsibly.
